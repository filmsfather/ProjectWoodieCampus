1) 레이아웃 시스템(기초 규칙)

컨테이너 규격: max-width 단계(모바일·태블릿·데스크톱)와 좌우 패딩 값을 디자인 토큰으로 고정.

그리드 프리미티브:

“Stack(세로 간격)”, “Inline(가로 정렬)”, “Grid(카드 자동배치)”, “Cluster(칩/태그 모음)” 같은 레이아웃 컴포넌트 추상화.

카드/위젯은 세로 스트레치(align-stretch) 기본, 내부 간격은 8pt 스케일.

자동 배치: 카드형 영역은 열 수 자동 조정(auto-fit / minmax 개념)과 행 높이 균등화(equal height) 규칙을 갖는다.

반응형 브레이크포인트: 최소 4단계(xs/sm/md/lg) 이름을 토큰화(색·간격처럼 “설계 언어”로).

2) 내비게이션/레이아웃 전환(적응형)

헤더/사이드바/바텀바 전환:

데스크톱: 상단 헤더 + 좌측 사이드바

모바일: 상단 헤더 + 바텀 탭/드로어(사이드바는 숨김)

공통 레이아웃 쉘: Layout.tsx가 **역할 탭(학생/교사/관리자)**과 경로에 맞춰 영역만 교체, 쉘은 항상 동일.

고정 요소: 스티키 헤더·하단 CTA 바는 세이프 에어리어(safe-area-inset) 대응.

3) 타이포·스페이싱(크기 자동화)

유동형 타이포그래피: 제목/본문 크기를 뷰포트에 따라 부드럽게 스케일(최소/최대 가드 포함).

8pt 스페이싱 스텝: 모든 마진/패딩을 토큰으로만 사용(임의 수치 금지) → 일관 정렬.

컴포넌트 내부 간격 규칙: 카드 헤더/본문/푸터 간격 표준화(예: 12/16/12).

4) 카드/대시보드 자동정렬 세부

카드 최소폭/최대폭 토큰화: “카드 S/M/L” 규격으로 정의 → 열 수 자동 계산.

높이 전략:

같은 행의 카드는 제목·요약 길이 차이가 있어도 하단 버튼이 같은 라인에 오도록 내부 레이아웃 고정.

콘텐츠 초과 처리: 두 줄 이상은 ellipsis·툴팁 규칙(타이틀 12줄, 설명 23줄).

5) 미디어·이미지 대응

반응형 이미지 규칙: 썸네일/커버는 고정 비율(aspect-ratio) + 오브젝트 크기 규칙(cover/contain).

지연 로딩/프리패치 정책: Fold 위·아래 이미지/폰트/데이터의 로드 우선순위 명세.

6) 폼/테이블/리스트(입력·데이터 뷰)

폼 레이아웃: 라벨-필드 정렬 규칙(모바일=스택, 데스크톱=2열 그리드), 오류/도움말의 위치 고정.

리스트/표:

모바일: 카드 리스트(세로),

데스크톱: 테이블+고정 헤더.

스켈레톤/로딩 상태: 로딩·빈 상태·오류 상태 세 가지 상태 UI를 각 컴포넌트에 포함.

7) 접근성·키보드·포커스 흐름

포커스 순서 규칙: 헤더 → 본문 → 사이드 → 푸터, 스킵 링크 제공.

키보드 내비: 탭 이동 가능, 드로어/모달은 포커스 트랩 + ESC 닫기.

명도 대비/상태 색: 브랜드 컬러 대비 기준(AA) + 상태색(성공/경고/위험) 토큰.

8) 성능·리플로우 안정성

컨텐츠 셰이핑: 이미지/차트/카드에 미리 크기 힌트를 주어 레이아웃 점프(CLS) 최소화.

가상 스크롤: 문제 리스트/로그 등 대용량 목록은 스크롤 가상화.

코드 스플리팅: 라우트·역할 단위 청크, 프리페치 규칙(다음 화면 예측).

9) 국제화·텍스트 가변성

문장 길이 대비: 버튼/카드 타이틀은 여유 폭 가드 + 2줄 허용, 레이아웃 붕괴 방지.

숫자/날짜 포맷: 자리수 늘어나도 그리드가 깨지지 않도록 우측 정렬/고정 폭 규칙.

10) 에러/빈 상태/권한

권한별 가시성: 버튼/카드가 숨겨질 때도 그리드 갭이 무너지지 않게 자리 재배치.

빈 상태 일관 템플릿: 아이콘·문구·CTA 패턴 통일(대시보드/목록/카드 공통).

11) 운영·테마·토큰

디자인 토큰 레지스트리: 색상·간격·반경·글꼴·섀도우를 CSS 변수로 중앙 관리.

역할별 테마 전환: <body data-role=\"student|teacher|admin\">에 따라 토큰만 바꿔 동일 레이아웃, 다른 분위기.

12) 테스트·검증 체계(코드 없이 정의)

뷰포트 매트릭스: 360/390/768/1024/1280/1440 등 표준 폭에서 화면 캡처 비교.

시나리오 테스트:

카드 수가 1개/3개/12개일 때 자동 정렬,

제목이 매우 길 때,

이미지 미로딩/에러일 때,

권한으로 카드 일부만 보일 때.

시각 회귀: 대시보드 핵심 섹션 스냅샷 기준선 마련(디자인 변경 시 레이아웃 이탈 탐지).
